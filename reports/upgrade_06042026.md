## Project Understanding

MiningResume is a dual-engine resume parser that converts PDF/DOCX/TXT files into structured JSON. It targets both academic and professional audiences. The LLM engine (`src/llm_based/`) is the primary focus — a SOLID-principled, v2.0 production-grade system with pluggable providers (HuggingFace, Ollama), caching, retry, and metrics. The rule-based engine (`src/rule_based/`) is a lighter XML-driven regex pipeline. A nascent knowledge-graph layer (`src/knowledge_base/`) connects parsed output to Neo4j.

## 🚨 Critical Issues (Runtime Bugs)

All items below were confirmed by reading actual file content. All will cause `AttributeError` or silent wrong behaviour at runtime.

| # | File | Line(s) | Issue |
|---|------|---------|-------|
| C1 | `src/llm_based/services/cache_service.py` | 66 | `settings.ttl_seconds` does not exist; correct attribute is `settings.cache_ttl_seconds`. Raises `AttributeError` every time `cache.set()` is called without an explicit TTL argument. |
| C2 | `src/llm_based/config/settings.py` | 34, 48, 62 | `bool(os.getenv("KEY", True))` is always `True` even when the env var is set to `"False"`, because `bool("False") == True`. Affects `USE_LOCAL_LLM`, `CACHE_ENABLED`, and `FAIL_FAST_FOR_BATCH`. |
| C3 | `src/llm_based/config/settings.py` | 73–76 | `prompts_file` is assigned `Field(default_factory=...)` but `Settings` is a plain Python class, not a Pydantic `BaseModel`. At runtime `settings.prompts_file` is a `FieldInfo` object, not a `Path`. Calling `.exists()` on it in `llm_service.py:59` always fails, so attribute-specific prompts are never loaded from YAML. |
| C4 | `src/llm_based/core/models.py` | 67–68 | `ExtractedResume.__getitem__` and `__setitem__` reference `self.fields`, which does not exist on a Pydantic model. Raises `AttributeError` if either method is called. |
| C5 | `src/llm_based/core/models.py` | 75 | `ExtractedResume.to_json()` calls `self.to_dict()`, which is never defined. Raises `AttributeError`. Should use `self.model_dump()`. |
| C6 | `src/main.py` | 47 | `run_llm_based(file_path: Path)` checks `FILE_PATH.exists()` (the module-level constant) instead of the received `file_path` argument, making the parameter useless. |

## ✅ What's Already Good

- Clean interface hierarchy in `core/interfaces.py` — all five interfaces (`IDocumentReader`, `ITextExtractor`, `ILLMProvider`, `ICacheService`, `IParserService`) are well-defined
- Thoughtful exception hierarchy with `LLMTimeoutError` and `LLMRateLimitError` as subtypes
- Thread-safe `InMemoryCacheService` with per-key TTL and `cleanup_expired()`
- `retry_with_backoff` decorator is clean, reusable, and has an async variant
- Structured JSON logging with rotating file handler
- Prompt externalization in `prompts.yaml` with per-attribute overrides — no prompt strings hardcoded
- `TXTTextExtractor` tries multiple encodings gracefully before raising
- `knowledge_base/graph_builder.py` is a clean, well-organized first pass at Neo4j integration
